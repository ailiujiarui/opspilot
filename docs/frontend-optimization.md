# OpsPilot 前端优化设计

## 1. 当前问题

### 1.1 产品壳层

- 72px 图标侧栏信息不足，角色选择器过窄，中文角色名称容易拥挤。
- 各模块仍保留自己的品牌栏，形成“侧栏 + 模块顶栏 + 页面标题”三层重复导航。
- 角色切换隐藏导航，但切换后若当前路由不再可访问，没有自动回到角色首页。
- 工作台中的关注数字和最近活动是静态演示，没有说明数据口径。

### 1.2 Agent 查询

- 筛选条件直接展示内部字段和操作符，例如 `budget gt 50000`，不适合业务用户。
- 数据引用按钮没有明确的选中反馈，也不会自动滚动到目标记录。
- 多轮条件只在内存中合并，没有清晰展示“上一轮条件”和“本轮新增条件”。
- 查询、澄清、错误、计划和执行结果共用同一种横条样式，信息层级不够清楚。
- 执行计划没有独立确认区，用户容易把“查询完成”和“即将写数据”混淆。

### 1.3 数据工作台

- 模块内部仍有独立品牌栏和账户头像，与统一壳层重复。
- 表头可点击性不明确，字段菜单和 Fields 按钮目前是视觉占位。
- 行选择 checkbox 只在 hover 出现，键盘和触屏用户难以发现。
- Agent 查询只载入前 100 条，但没有分页/继续加载入口。
- 表格内部使用英文状态值作为数据协议，UI 映射正确，但筛选条件解释没有映射。

### 1.4 自动化编辑器

- 节点文案已领域化，但内部 kind 和配置表单仍是通用文本框。
- 节点库提示“拖入”，当前实际交互主要是点击添加。
- 模块存在第二套顶栏、头像和保存状态，与统一壳层重复。
- 校验结果只在右侧底部出现，缺少可定位的错误列表。
- 运行状态只显示单节点结果，没有清晰的步骤时间线。

## 2. 优化目标

1. 保持安静、密集、可扫描的企业工作台风格。
2. 让招聘方在 30 秒内理解“风险查询 -> 数据证据 -> 计划确认 -> 固定流程 -> 审计”。
3. 清楚区分只读查询与写操作确认。
4. 提升 1366x768 和 1920x1080 下的信息利用率。
5. 保持现有状态逻辑，不在视觉优化阶段引入新业务模块。

## 3. 统一布局

### 3.1 左侧导航

- 桌面端宽度从 72px 调整为 184px，显示图标和完整中文标签。
- 品牌区只显示一次：`OpsPilot` 与辅助说明“AI 项目交付工作台”。
- 角色切换移到底部用户菜单，显示当前角色全称。
- 1366px 仍显示完整导航；小于 900px 收起为 60px 图标栏。

### 3.2 模块页头

删除数据页和自动化页内部的第二套品牌栏。统一为：

```text
面包屑 / 页面标题 / 页面说明                      保存状态 / 主要操作
```

页面标题使用 20 至 24px，内部工具栏使用 12 至 13px，不使用营销式大标题。

### 3.3 视觉变量

```text
背景：#F4F6F8
表面：#FFFFFF
正文：#20252B
次要文字：#6F7983
边框：#DCE1E5
品牌色：#4F5FD7
成功：#2E8053
警告：#A36A13
错误：#B6414D
圆角：4px / 6px
基础间距：4px
```

不增加渐变、装饰球和大面积品牌色背景。

## 4. Agent 结果面板

Agent 面板拆为四种明确状态：

### 4.1 输入状态

- 单行输入、常用查询建议和“只读查询”标记。
- Enter 提交，Esc 清空。
- 查询期间禁用重复提交，并显示当前阶段。

### 4.2 查询解释

内部协议映射为业务语言：

```text
budget gt 50000 -> 预算高于 ¥50,000
status eq Active -> 状态为进行中
due lt 2026-07-16 -> 截止日期早于 2026-07-16
owner eq 陈梅 -> 负责人为陈梅
```

明确标注本轮新增条件，支持删除单个条件后重新查询。

### 4.3 数据证据

- 展示总命中数、当前载入数和数据更新时间。
- 引用显示记录 ID、项目名称和命中字段。
- 点击引用后滚动至对应虚拟行并高亮，而不只改变 Store 中的 active 值。

### 4.4 写操作确认

执行计划使用独立的确认面板：

- 黄色警示图标。
- 影响记录数。
- 变更字段和前后值。
- 计划过期时间。
- “返回查询”和“确认执行”两个明确操作。
- 执行中禁止重复点击。

## 5. 数据表格优化

- 保留固定 40px 行高和虚拟滚动。
- 行 checkbox 默认可见但降低对比度；选中后增强。
- 表头字段菜单只在功能真实可用时展示；删除无行为的占位按钮。
- 增加“载入更多”或游标分页状态，明确浏览器没有一次加载全部数据。
- 表格焦点与 Agent 引用定位联动。
- 保存状态移入统一页头，不保留重复头像。
- 只读/编辑状态用文字和图标共同表达。

## 6. 自动化编辑器优化

- 删除重复品牌栏，画布直接占据模块可用区域。
- 节点库提示改为“点击添加到画布”，与真实行为一致。
- 为领域节点提供专用配置字段，不使用统一自由文本：
  - 查询逾期项目：状态、截止日期规则。
  - 检查项目风险：风险阈值和分支。
  - 按负责人分组：分组字段。
  - 等待人工确认：确认角色和超时时间。
  - 记录处理结果：审计事件类型。
- 校验结果形成可点击列表，点击后定位节点。
- 试运行展示步骤时间线：等待、运行、成功、失败。

## 7. 响应式要求

### 1366x768

- 完整侧栏 184px。
- 数据页 Agent 面板最大占用两行。
- 表格至少显示 10 行数据。
- 自动化节点库 220px、属性面板 240px，画布保持主要宽度。

### 1920x1080

- 页面内容不无限拉伸，工作台内容最大宽度 1440px。
- 数据表格可使用全部剩余宽度。
- 工作台 Agent 区与风险摘要保持合理比例。

### 小于 900px

- 侧栏收为图标栏。
- 自动化左右面板改为抽屉。
- 数据表格允许横向滚动，不强行压缩固定列。

## 8. 实施顺序

1. 重构统一壳层与模块页头，清除重复品牌和头像。
2. 重构 Agent 状态面板与业务语言条件映射。
3. 完成记录引用滚动定位和高亮。
4. 优化表格选择、无行为控件和载入状态。
5. 优化自动化编辑器布局、专用字段和运行时间线。
6. 在 1366x768、1920x1080 和 900px 窄屏下人工检查。
7. 运行 lint、测试、构建和完整 Code Review 后再推送。

## 9. 验收标准

- 页面只出现一个主品牌 OpsPilot。
- 角色切换后不会停留在无权访问的路由。
- Agent 的查询、澄清、错误、计划和完成状态视觉可区分。
- 所有筛选条件使用业务中文表达。
- 点击数据引用可以定位并高亮真实表格行。
- 没有无行为的主要按钮和菜单。
- 1366x768 下核心操作不重叠，表格至少显示 10 行。
- 自动化画布使用领域节点和对应配置，不是通用文本 Demo。
- 前端 lint、TypeScript、API 测试和生产构建通过。
